# NewRAG 前端配套方案（详细规划）
版本：v1.0  
日期：2026-01-05  
范围：前端界面与交互、评测展示与后端接口对接  

---

## 1. 目标与非目标

### 目标
- 🎯 构建可验证的前端体验闭环：上传文档 → 结构浏览 → 问答 → 证据可视化 → 反馈与评测。
- 🎯 与后端状态机对齐，前端能够显示解析、索引、检索、融合与校验过程。
- 🎯 提供清晰的证据链与引用路径，支持图谱与结构树联动。
- 🎯 输出可追溯评测报告与运行状态，便于验收与复盘。

### 非目标
- 不实现后端算法与模型推理逻辑。
- 不在前端存储 LLM 明文密钥或敏感配置。
- 不提供多模态渲染能力，仅保留扩展空间。

---

## 2. 用户角色与核心场景

### 角色
- 管理员：上传与维护文档、触发索引、查看系统健康与评测结果。
- 研究与标注人员：提问、查看证据、提交反馈与证据标注。
- 访客用户：只读查询与结果查看（可选）。

### 核心场景
1. 文档上传与结构树校验。
2. 问答检索与证据链查看。
3. 证据冲突标记与反馈提交。
4. 评测任务触发与报告导出。

---

## 3. 信息架构与页面规划

### 页面清单
- 问答工作台：提问、参数设置、结果与证据展示。
- 文档管理：上传、列表、详情与结构树浏览。
- 索引管理：构建与状态查询、版本信息。
- 证据图谱：证据链图谱与结构树联动。
- 评测与报告：评测任务、指标展示、报告导出。
- 系统状态：健康检查、任务队列状态与版本信息。

### 导航与路由
- 桌面端：左侧主导航 + 顶部状态区。
- 移动端：底部 Tab + 抽屉导航。
- 路由结构建议：`/qa`、`/docs`、`/docs/:id`、`/index`、`/graph`、`/eval`、`/status`。

---

## 4. 前端架构与目录建议

### 技术栈建议
- 框架：React + TypeScript（可替换为 Vue）。
- 构建：Vite。
- 状态管理：轻量全局状态 + 请求缓存。
- 图谱渲染：Canvas 或 SVG 方案，支持缩放与拖拽。

### 目录结构建议
```
src/
  pages/          # 页面级组件
  components/     # 通用组件
  services/       # API 请求与数据适配
  stores/         # 状态与缓存
  layouts/        # 布局与导航
  utils/          # 工具与格式化
  assets/         # 图标与静态资源
```

## 5. 状态模型与前端状态机

### 文档状态映射
- `RAW` → 已上传，待解析
- `PARSED` → 结构树可用
- `CHUNKED` → chunk 可查
- `EMBEDDED` → 向量生成完成
- `INDEXED` → 索引可用
- `READY` → 可检索

### 查询状态映射
- `RETRIEVE_COARSE` → 粗检索进行中
- `RERANK_FINE` → 精排进行中
- `FUSE_EVIDENCE` → 证据融合进行中
- `GENERATE_ANSWER` → 生成答案中
- `VERIFY_ANSWER` → 一致性校验中
- `DONE` → 结果就绪

### UI 状态
`Idle` / `Loading` / `Empty` / `Error` / `Success`，各页面统一状态提示。

## 6. 数据流与缓存策略

### 问答流程
1. 提交 query → 显示步骤进度条。
2. 返回 answer + evidence + graph → 渲染证据与引用。
3. 校验状态显示并允许反馈。

### 文档流程
1. 上传文档 → 显示解析与索引状态。
2. 结构树浏览 → 与证据链联动定位。

### 缓存策略
- 文档列表与详情缓存 5 分钟，可手动刷新。
- 问答结果按 query 哈希缓存，默认 10 分钟。
- 评测报告缓存到本地存储，带版本号。

## 7. API 对接要点与数据契约

### 文档与索引
- `POST /api/documents/upload`：上传文件，返回 `doc_id` 与状态。
- `GET /api/documents`：列表分页与状态筛选。
- `GET /api/documents/{id}`：文档详情与元信息。
- `GET /api/documents/{id}/tree`：结构树数据。
- `POST /api/index/build`：触发索引构建。
- `GET /api/index/status`：索引状态与版本。

### 问答与证据
- `POST /api/qa/query`：入参 `query` `top_k` `rerank_k`，出参 `answer` `evidence[]` `citations[]` `graph`。
- `GET /api/answers/{id}/evidence`：证据链与评分。
- `GET /api/answers/{id}/graph`：图谱结构 `nodes[]` `edges[]`。

### 反馈与配置
- `POST /api/feedback`：`node_id` `score` `comment`。
- `POST /api/llm/config`：仅保存连接配置，不保存明文密钥。
- `POST /api/llm/test`：连通性测试结果展示。
- `/health` 与 `/ready`：前端健康状态展示与告警。

## 8. 证据展示与结构树联动

### 证据卡片
- 展示字段：`chunk_id`、`score`、`path`、`source_rank`。
- 显示冲突与冗余标记，支持快速过滤。

### 图谱视图
- `nodes[]`：`id` `type` `title` `path` `score`。
- `edges[]`：`source` `target` `relation`。
- 支持缩放、拖拽、点击高亮与定位到结构树。

### 结构树联动
- 点击证据 → 定位结构树节点。
- 点击结构树节点 → 高亮证据列表与图谱节点。

---

## 9. 关键交互与页面细节

### 问答工作台
- 支持参数：`top_k`、`rerank_k`、结构先验开关。
- 展示检索阶段与耗时，显示校验状态。
- 结果区支持复制、导出与引用路径展示。

### 文档管理
- 拖拽与批量上传，显示进度与失败原因。
- 文档状态可视化，支持按状态筛选。
- 结构树支持展开收起与快速搜索。

### 反馈与标注
- 评分、评论、冲突标记与不确定说明。
- 可选择证据片段用于标注导出。

---

## 10. 视觉与交互规范

### 视觉基调
- 基调：浅米白背景 + 深墨绿文字，强调可靠与学术感。
- 强调色：暖橙用于提示与交互高亮。
- 字体建议：标题使用思源宋体，正文使用思源黑体。

### 动效与过渡
- 页面加载：轻量渐入 + 分段出现。
- 证据卡片：排序变化使用平滑过渡。
- 图谱：缩放与拖拽保持 60fps。

---

## 11. 性能与体验基线

- 首屏渲染 ≤ 2s（本地缓存命中场景）。
- 长列表使用虚拟滚动。
- 图谱节点超过阈值时自动降采样。
- 关键操作全程提供骨架屏与加载提示。

---

## 12. 错误处理与降级

- LLM 超时：展示“仅证据摘要”与重试入口。
- 索引不可用：提示索引状态并提供构建入口。
- 网络异常：保留当前输入与草稿。

---

## 13. 评测与验收展示

### 指标展示
- 检索：Recall@N、MRR、NDCG。
- 生成：引用覆盖率、一致性通过率。
- 融合：冗余率、冲突率。

### 报告导出
- JSON 与 CSV 导出，包含版本号与时间戳。
- 单次评测运行提示 < 60s 约束。

## 15. 风险与待确认事项

### 风险
- 图谱数据量过大导致渲染卡顿。
- 后端字段变动导致展示异常。
- 证据链过长导致阅读负担过高。

### 待确认
- 前端框架最终选型。
- 图谱渲染技术与性能方案。
- 是否引入权限系统与登录页。
